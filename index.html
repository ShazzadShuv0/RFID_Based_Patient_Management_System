Project Introduction

An overview of our innovative hospital monitoring system

The RFID BASED PATIENT MONITORING SYSTEM presents the design, simulation and implementation of a hand held RFID-based patient monitoring system. In many hospitals across Bangladesh, patient management still relies on handwritten records and manual data entry which of course leads to frequent errors and inefficiencies in treatment.

A study conducted in private hospitals in Bogra revealed an average of 3.46 prescription errors per patient including illegible handwriting, missing dosage details and serious drug interactions. And these problems increased treatment costs, extended hospital stays as well as sometimes becoming life threatening outcomes. These issues highlight the urgent need for an automated system that ensures accurate and secure handling of patient information.

The RFID Based Patient Monitoring system addresses this problem by using RFID tags to uniquely identify each patient and provide doctors with instant access to medical history, prescriptions and test results. This system will reduce human error, enhance co-ordination between doctor, hospital staff and enable continuous patient monitoring.

By integrating all patient data into a central and accessible database, this project aims to improve treatment accuracy, operational efficiency and record security which will ultimately contribute to safer and smarter healthcare management across Bangladeshi hospitals.

Project Scope

System specifications define the functional, electrical and performance requirements that the RFID-Based Patient Monitoring System must satisfy to ensure reliable, safe and efficient operation in a hospital environment.

Functional Specifications

  • Uniquely identify patients using passive 125 kHz RFID tags.
  • Retrieve and display patient medical information through a Doctor’s Handheld Unit (DHU).
  • Allow authorized updates to patient data.
  • Operate in both online (cloud-connected) and offline modes.
  • Ensure persistent storage of patient records after power loss.

Electrical and Power Specifications

  • Power Source: Three 18650 Li-ion batteries
  • Charging Mode: 1S3P (parallel)
  • Service Mode: 3S1P (series)
  • Nominal Output Voltage: 5 V DC
  • Maximum Load Current: ~732 mA
  • Voltage Regulation: LM2596 buck converter (≈85% efficiency)
  • Total Energy Capacity: 10.545 Wh

Hardware Specifications

  • Microcontroller: ESP32 DevKit V1 (3.3 V logic, dual-core, Wi-Fi enabled)
  • RFID Module: RDM6300 (125 kHz, UART interface)
  • Display: SPI-based TFT LCD (240 × 320 resolution)

Communication and Data Specifications

  • RFID Communication: UART (9600 baud)
  • Display Interface: SPI
  • Cloud Communication: Wi-Fi using Blynk IoT platform
  • Data Storage: JSON-based files stored in ESP32 flash memory using LittleFS

Performance Specifications

  • RFID detection time: < 1 second
  • Estimated continuous runtime: ~2.45 hours (worst-case load)
  • GUI response: Real-time and lag-free
  • Data retention: Non-volatile storage

PCB and Mechanical Specifications

  • Subsystem-01 (Power Supply and Distribution Unit): 2500 × 2700 mil
  • Subsystem-02 (Doctor’s Handheld Unit): 2500 × 3745 mil
  • Overall hardware dimensions :
  • Height: 2 inches
  • Length: 6.5 inches
  • Width: 4.5 inches

Methodology

When the Doctor’s Handheld unit (DHU) touches the RFID tag attached to a patient’s bed, the RFID system reads the bed number and sends it to the cloud. Using this bed number, the cloud system identifies the patient assigned to that bed and retrieves the patient’s medical history and test reports. This information is then displayed on a screen for the doctor to review. Based on these details, the doctor prescribes the required medicines or treatment, and the new prescription and updates are saved back to the cloud for future use.
